## 1. 지원서 항목 ① — 해결하려는 생활 속 불편함 (문제 정의)

**"읽을 수는 있지만, 이해할 수 없는 서류들"**

어르신들의 우편함에는 매달 건강보험 고지서, 재산세 통지서, 복지 안내문이 도착합니다. 글자를 못 읽는 것이 아닙니다. "체납 시 가산금이 부과됩니다", "구비서류를 지참하여 관할 기관에 방문하시기 바랍니다" — 행정용어와 한자어의 벽 앞에서, 읽고도 무엇을 해야 할지 알 수 없는 것입니다.

결과는 세 갈래로 나타납니다.
1. 기한을 놓쳐 **가산금**을 물거나,
2. 받을 수 있는 **혜택을 몰라서** 놓치거나,
3. 진짜 고지서와 가짜 독촉장을 구분하지 못해 **사기 피해자**가 됩니다.

기존 해법의 빈자리: 설리번A 같은 시각보조는 문서를 '소리 내어 읽어'주지만 행정용어 이해까지 도와주진 않습니다. 웰로 같은 정책 추천은 사용자가 '찾아가야' 하지만, 어르신의 문제는 이미 '도착한 서류' 앞에서 시작됩니다. 지금까지 이 빈자리는 자녀·손주가 곁에 있을 때만 메워왔습니다.

**'효자손'의 재정의**: 어르신에게 필요한 것은 읽기 대행이 아니라 **"이게 무슨 서류이고(이해) → 그래서 뭘 해야 하고(행동) → 믿어도 되는지(안전)"**에 답하는 손주의 역할입니다. 사진 한 장으로 이 세 질문에 답하는, 곁에 없는 손주를 대신하는 AI입니다.

## 2. 지원서 항목 ② — AI 활용 방식

**포지셔닝: 범용 서류 해석기** — 고지서·약관·계약서·문자 안내·병원/약 안내·신청서·복지 안내 등 어르신이 일상에서 마주치는 **모든 문서·문자**를 사진 한 장으로 해석합니다. 사기 지킴이는 그중 대표 차별화 기능입니다. 엔진은 문서 유형을 가리지 않는 범용 프롬프트이며, 유형별로 `핵심주의`(약관 독소조항·위약금·자동연장 / 동의서 개인정보 범위 / 복약 주의 / 문자 확인포인트)를 맞춤 추출합니다.

효자손은 **서류 사진 1장**을 입력받아 3가지 답(이해·행동·안전)을 생성하며, 사기 판별은 **5개 층이 협력하는 방어 체계**로 구현했습니다. 단일 AI 호출이 아니라, AI 판단 위에 규칙·공공데이터·사실확인을 겹쳐 신뢰도를 높였습니다.

### 2-1. 비전 AI (문서 이해·행동·안전 1차 판정)
- **주 모델**: OpenAI **gpt-5.4-nano** (이미지 입력 지원, Structured Outputs로 JSON 스키마 강제 → 파싱 실패 없음, API 입력 기본 미학습)
- **폴백 모델**: Google **Gemini 2.5 Flash** (주 모델 실패 시 자동 전환)
- **이중화 의의**: 서로 다른 회사의 모델로 이중화 → 한쪽 장애·정책 변경에도 견고 (모델 어댑터 패턴 — `GEMINI_API_KEY` 유무로 폴백 자동 구성)
- 존댓말 + 초등 5학년 어휘 + 한자어→일상어 변환을 프롬프트로 강제

### 2-2. 사기 지킴이 — 5개 층 (핵심 차별화)
| 층 | 역할 | 근거 데이터/기술 |
|---|---|---|
| **1층 — 규칙 엔진** | 정규식·휴리스틱으로 스미싱·사칭·미끼 패턴 즉시 포착 (클라이언트) | KISA 보호나라 수칙 규칙화 |
| **2층 — 블랙리스트** | 서류/QR 속 URL을 실제 피싱 명단과 대조 → 걸리면 위험 확정 | **KISA 피싱사이트 URL 27,582건**(공공데이터포털 odcloud), Upstash(KV) 캐시, Vercel Cron 일 1회 갱신 |
| **3층 — 비전 AI** | 명단에 없는 신종 수법을 문맥으로 감지 | gpt-5.4-nano / Gemini |
| **사실확인** | 서류 속 기관·업체·금액을 공공데이터로 대조 | 국세청 사업자등록 진위, 국토부 실거래가(전국 286개 시군구), 금감원 DART(참고) |
| **게이트+역확인** | 돈·행동을 요구하는 공적 문서면 최소 '주의'로 올리고 **공식 대표번호로 직접 확인** 유도 | 기관 17곳 공식번호 디렉터리 |

- 최종 위험도는 세 층 중 가장 위험한 판정을 채택(격상형). AI가 '안전'이라 해도 규칙·블랙리스트·게이트가 위험을 끌어올릴 수 있음.
- 결과는 **신호등(🟢 안전 / 🟡 주의 / 🔴 위험)** + 위험 시 "가족에게 먼저 물어보세요" + 신고 3종(**경찰 112 · 금융감독원 1332 · 스미싱 118**) 안내. 발신기관을 알면 해당 기관 공식 대표번호도 함께 제시

### 2-3. QR 코드 대응 (신종 수법)
가짜 고지서가 링크 대신 **QR코드**로 결제·개인정보 입력을 유도하는 수법에 대응. 서버가 이미지 속 QR을 **디코딩만 하고(절대 접속 안 함)** URL을 뽑아 2층 블랙리스트로 대조. 저화질 실패 시 조용히 넘어가며, 어르신은 아무 조작도 필요 없음(사진만 찍으면 됨).

### 2-4. 부가 기능 (기획문서 F1~F5)
- **쉬운말 풀이**: 한 줄 요약(25자) + 쉬운 설명(3~4문장) + 어려운 낱말 풀이
- **할 일 짚어주기**: 기한/장소/금액/준비물 체크리스트
- **TTS 음성 읽기**: Web Speech API(ko-KR), 외부 의존성·비용 0
- **지난 서류 다시 보기**: IndexedDB에 결과 텍스트만 저장(사진 원본 미저장)
- **결과 카드**: Canvas로 세로형(9:16) 이미지 생성 → 사진 저장 & 가족 공유(원본 동봉 옵트인)

## 3. 지원서 항목 ③ — 핵심 AI 프롬프트 (기재용 확정본)

```
당신은 어르신의 다정한 손주 역할을 하는 AI입니다.
첨부된 서류 사진을 분석해 아래 JSON으로만 답하세요.

[말투 규칙]
- 존댓말, 초등학교 5학년이 이해할 어휘만 사용
- 한자어·행정용어는 반드시 일상어로 바꿈
  (예: "납부하다"→"돈을 내다", "과태료"→"벌금")

[출력 형식]
{
 "문서종류": "", "한줄요약": "25자 이내", "쉬운설명": "3~4문장",
 "할일": [{"내용":"", "기한":"", "장소":"", "준비물":""}],
 "낱말풀이": [{"어려운말":"", "쉬운말":""}],
 "위험도": "안전|주의|위험", "위험이유": "",
 "URL목록": ["서류에 보이는 인터넷 주소, 없으면 빈 배열"],
 "사업자번호": "10자리 숫자만, 없으면 빈 문자열",
 "발신기관": "", "부동산거래": "매매|전세|월세|빈 문자열",
 "부동산지역": "", "부동산금액": "만원 단위 숫자만"
}

[안전 규칙]
- 다음 중 하나라도 뚜렷하면 위험도를 반드시 "위험"으로:
  · 수사기관·정부기관·공공기관·금융회사 사칭(가짜/위조 공문 포함)
  · 링크 접속 유도 · 개인정보(주민번호·계좌·비밀번호·인증번호) 입력 요구
  · 국민지원금·환급금·당첨·대출 등을 미끼로 한 링크 유도
  · "오늘까지·즉시·구속·압류·과태료" 등 겁을 주는 문구
- 애매하면 "주의", 안전 신호만 있으면 "안전"
- 주민등록번호·계좌번호는 출력에서 일부 마스킹(예: 800101-1******)
- 불확실하면 추측 말고 "가까운 주민센터나 해당 기관에 확인하세요"를 할일에 포함
- 의료·법률 판단은 단정하지 않음
```
> 실제 운영 프롬프트는 `api/analyze.js`의 `SYSTEM_PROMPT`. OpenAI는 위 스키마를 Structured Outputs(strict)로 강제.

### 5-4. 통합 테스트 결과 (실제 이미지 9종, 5개 층 전부 on)
- 사기 5종(택배 스미싱·국민지원금·위조 식약처 공문×2·QR 가짜 범칙금·정부24 사칭) → 모두 **위험** 판정
- 정상 3종(건강보험 확인서×2·정상 QR) → **주의/안전** (오탐 없음)
- 위조 공문 표식有: 서버 '주의' → 1층 규칙이 '위조 공문' 포착 → **위험 격상** 확인

8. **QR 안전**: 디코딩만, 절대 자동 접속 안 함
9. **출처 표기**: KISA, 공공데이터포털, 금감원 DART, 사용 라이브러리(jsQR 등)
