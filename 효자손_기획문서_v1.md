## 1. 문제 정의 (지원서용 서술 확정본)

**"읽을 수는 있지만, 이해할 수 없는 서류들"**

어르신들의 우편함에는 매달 건강보험 고지서, 재산세 통지서, 복지 안내문이 도착합니다. 글자를 못 읽는 것이 아닙니다. "체납 시 가산금이 부과됩니다", "구비서류를 지참하여 관할 기관에 방문하시기 바랍니다" — 행정용어와 한자어의 벽 앞에서, 읽고도 무엇을 해야 할지 알 수 없는 것입니다. 결과는 세 갈래로 나타납니다. 기한을 놓쳐 가산금을 물거나, 받을 수 있는 혜택을 몰라서 놓치거나, 진짜 고지서와 가짜 독촉장을 구분하지 못해 사기 피해자가 됩니다. 국민의 85%가 정책 대상자임에도 자신이 대상자인지 모르거나 절차가 복잡해 혜택을 누리지 못하고 있으며(출처: 웰로 서비스 소개 — 지원서에는 과기정통부 '디지털정보격차 실태조사' 고령층 수치로 교체·보강 권장), 이 격차는 디지털에 익숙하지 않은 고령층에서 가장 큽니다.

기존 해법에는 빈자리가 있습니다. 설리번A 같은 시각보조 서비스는 문서를 '소리 내어 읽어'주지만, 행정용어를 읽어준다고 이해가 되지는 않습니다. 웰로 같은 정책 추천 서비스는 사용자가 '찾아가야' 하지만, 어르신의 문제는 이미 '도착한 서류' 앞에서 시작됩니다. 그리고 지금까지 이 빈자리는 자녀와 손주가 메워왔습니다 — 곁에 있을 때만.

'효자손'은 이 문제를 다시 정의합니다. 어르신에게 필요한 것은 읽기 대행이 아니라, **"이게 무슨 서류이고(이해), 그래서 뭘 해야 하고(행동), 믿어도 되는지(안전)"**에 답해주는 손주의 역할입니다. 사진 한 장으로 이 세 가지 질문에 답하는 AI — 곁에 없는 손주를 대신하는, 우리 곁의 착한 AI입니다.

### 할 일 (지원서 보강)
- [ ] 85% 수치 → 과기정통부 디지털정보격차 실태조사 고령층 디지털정보화 수준 수치로 교체/병기
- [ ] 고령층 보이스피싱·스미싱 피해 통계 1건 추가 (경찰청/금감원 보도자료)

## 2. 서비스 정의

- **서비스명**: 효자손 (부제: 서류 읽어주는 AI 손주)
  - 어르신에게 친숙한 물건 + '가려운 곳을 긁어주는 도구' + '효도하는 손주' 삼중 의미
- **한 줄 정의**: 서류 사진 한 장으로 이해(쉬운말) → 행동(할 일) → 안전(사기 판별)까지 안내하는 어르신 전용 AI 웹앱
- **타깃**: 디지털 취약 고령층(1차), 그들의 자녀·손주(2차 — 공유 수신자)
- **공모 분류**: 일상생활(정보격차 해소) + 지역사회(안전) 교차

|---|---|---|---|---|
| 대상 | 시각장애인 | 정책 탐색자 | 일반 사용자 | 고령층 |

## 3. 핵심 기능 명세

### F1. 쉬운말 풀이
- 서류 사진 1장 → ① 한 줄 요약(25자 이내) ② 쉬운 설명(3~4문장) ③ 어려운 낱말 풀이
- 초등 5학년 어휘, 존댓말
- TTS 음성 읽기: 브라우저 Web Speech API(ko-KR), 외부 의존성·비용 0
- 글자 크기 3단 조절(기본값을 크게)

### F2. 할 일 짚어주기
- 기한 / 장소 / 금액 / 준비물 추출 → "12월 15일까지 / 주민센터에 / 신분증 들고" 수준의 체크리스트
- 불확실하면 추측 금지, "가까운 주민센터나 해당 기관에 확인하세요" 항목으로 대체
- 심사 배점 최상위 '유용성 30점' 직격 기능

### F3. 사기 지킴이 (3층 이중화)
- **1층 — 규칙(클라이언트)**: 정규식·휴리스틱
  - 계좌번호 + 긴급/독촉 문구 조합, 단축 URL(bit.ly 등), 개인정보 입력 요구
  - KISA 공식 수칙의 규칙화: "정부기관·금융회사는 전화/문자로 원격제어앱 설치를 요구하지 않음" 등 (출처 표기: KISA 보호나라)
- **2층 — 블랙리스트(서버)**: 공공데이터포털 'KISA 피싱사이트 URL' 데이터
  - Vercel Cron → 주기 fetch → Edge Config/KV 캐시 → 분석 시 서류 내 URL 대조
  - 한계 명시: 연 단위 갱신 → 지원서 발전 방향에 "실시간 위협 인텔리전스 연동" 기재
  - 공공데이터 활용신청 필요(오픈API JSON/XML 자동변환 제공), 라이선스·출처 표기 준수
- **3층 — AI 판단**: 리스트에 없는 신종 수법을 문맥으로 감지
- 결과: 신호등(🟢 안전 / 🟡 주의 / 🔴 위험) + 위험 시 "가족에게 먼저 물어보세요 + 금융감독원 1332 / 스미싱 상담 118" 안내

### F4. 지난 서류 다시 보기 (기기 내 기록)
- IndexedDB에 **분석 결과 텍스트(JSON)만** 저장 (사진 원본 미저장 — 용량·위험 동시 감소)
- 목록: 날짜 + 문서종류 + 신호등 색 표시
- 개별 삭제 / 전체 삭제 버튼
- 원칙 문구: "내 기록은 내 휴대폰에만 보관됩니다"
- 한계: 기기 변경 시 소실 → 지원서에 "의도된 Privacy by Design"으로 서술, 발전 방향 "가족 연동 시 암호화 동기화"

### F5. 결과 카드 — 사진 저장 & 가족 공유
- Canvas API로 세로형(9:16) "결과 카드" 이미지 생성 (외부 라이브러리 불요)
- 카드 구성: 상단 신호등 색 배너(갤러리 썸네일에서도 보이도록 크게) → 한 줄 요약(최대 글씨) → 할 일 체크리스트 → 하단 날짜·서비스명. 주민번호류 마스킹 동일 적용
- **저장**: 버튼명 "📷 사진으로 저장하기" 단일화, 내부 분기
  - Android: 다운로드 → 갤러리 자동 노출
  - iOS: Web Share API로 이미지 전달 → 공유 시트 "이미지 저장" → 사진 앱 직행
- **가족 공유**: 동일 카드 이미지를 Web Share로 카톡 등에 전송
  - **원본 사진 동봉(신규 확정)**: Web Share Level 2 `files` 배열로 카드+원본 동시 전송 가능
    - 기본값: 카드만 / "원본 사진도 함께 보내기" **옵트인 체크박스**로만 원본 포함 (원본은 마스킹 불가하므로)
    - `navigator.canShare({files})` 사전 체크, 미지원 시 카드만 폴백
    - **세션 제약**: 원본 동시 공유는 분석 직후 화면에서만 가능(무저장 원칙상 원본은 세션 종료 시 폐기). '다시 보기'에서는 카드만 재생성·공유
- 부수 효과(지원서 서술): 갤러리 저장 → 구글포토/아이클라우드 자동 백업 → 서버 무저장 원칙을 지키면서 보관 문제 해결

### 공통 설계 원칙
- 회원가입 없음, 홈 → 결과까지 터치 2번 이내
- 심사용 통로: 홈에 **"샘플 서류로 체험하기"** 버튼 필수 (심사위원은 실물 고지서가 없음)

## 4. 화면 구성 (3화면)

1. **홈**: 화면 절반 크기 초대형 버튼 "📷 서류 사진 찍기" / 아래 "샘플 서류로 체험하기" / 하단 "지난 서류 다시 보기" 목록 진입
2. **결과**: 신호등 배너 → 한 줄 요약 → 할 일 체크리스트 → 쉬운 설명 + 낱말 풀이 / 하단 고정 바: [🔊 듣기] [가A 글자] [📷 사진으로 저장] [가족에게 보내기(+원본 동봉 체크)]
3. **도움말**: 그림 3컷 사용법 + "이 서비스는 사진을 저장하지 않아요" 신뢰 문구

디자인: 고대비, 최소 16px 이상 기본(노인 기준 상향), 터치 타깃 48px+, 애니메이션 최소화. frontend-design 스킬 적용해 템플릿 느낌 탈피.

## 5. 기술 아키텍처

```
[브라우저 PWA-lite]
  ├ 카메라/파일 입력 → 클라이언트 리사이즈·압축(고지서 사진 대형)
  ├ 1층 사기규칙(정규식) 
  ├ IndexedDB(결과 텍스트만) 
  ├ Canvas 결과 카드 생성 → 다운로드/Web Share(files[] 카드+원본)
  └ Web Speech API TTS(ko-KR)
        │ HTTPS POST (이미지 base64)
        ▼
[Vercel Serverless /api/analyze]
  ├ API 키 환경변수 은닉 (클라이언트 노출 금지)
  ├ Rate limit (IP당 시간당 N회) — 과금·남용 방지
  ├ 2층: KV/Edge Config의 KISA 피싱 URL 대조
  ├ 3층: 비전 모델 호출 → JSON 파싱 (실패 시 1회 자동 재시도)
  └ 응답 후 이미지 즉시 폐기 (완전 stateless, 로그에 이미지 미기록)

[Vercel Cron /api/refresh-blacklist]
  └ 공공데이터포털 KISA 피싱 URL fetch → KV 갱신 (일 1회)
```

### 비전 모델 (확정)
- **주 모델**: Gemini 2.5 Flash **유료 티어** + 월 지출 한도 설정
  - 사유: 무료 티어는 입력 데이터가 학습에 활용될 수 있음 → 개인정보 서류 서비스에 부적합. 유료는 데이터 비공개 + 호출당 1~2원 수준 + 지출 한도로 과금 폭탄 차단
  - 개발·테스트 단계는 무료 티어 + **샘플 서류만** 사용
- **폴백 모델**: OpenAI gpt-5.4-nano (API 입력 기본 미학습, Structured Outputs로 JSON 스키마 강제) — 주 모델 실패 시 전환. **서로 다른 회사 모델로 이중화** → 한쪽 장애·정책 변경에도 견고하다는 점을 지원서 어필
- 모델 어댑터 패턴으로 구현해 교체 용이하게

### 예외 처리
- 흐릿한 사진/텍스트 미검출 → "다시 찍어주세요" 큰 안내 + 촬영 팁
- API 실패 → 폴백 모델 → 그래도 실패 시 "잠시 후 다시 시도" (기술용어 없는 문구)
- JSON 파싱 실패 → 1회 재시도 후 오류 안내

```

[말투 규칙]
- 존댓말, 초등학교 5학년이 이해할 어휘만 사용

  (예: "납부하다"→"돈을 내다", "과태료"→"벌금")

[출력 형식]
{
 "문서종류": "",
 "한줄요약": "25자 이내",
 "쉬운설명": "3~4문장",
 "할일": [{"내용":"", "기한":"", "장소":"", "준비물":""}],
 "낱말풀이": [{"어려운말":"", "쉬운말":""}],
 "위험도": "안전|주의|위험",
 "위험이유": ""
}
```

## 7. 보안·개인정보 설계 (지원서 '안전' 항목 + 대회 유의사항 대응)

1. **비수집 원칙**: 회원가입 없음, 서버 저장 없음(stateless), 로그에 이미지·개인정보 미기록
2. **전송 보호**: HTTPS, 클라이언트 리사이즈로 최소 데이터 전송
3. **API 보안**: 키 환경변수 은닉, 서버리스 경유(클라이언트 직호출 금지), rate limit
4. **모델 계약**: 유료 티어 사용으로 입력 데이터 학습 미활용 보장
5. **출력 마스킹**: 주민번호류 자동 마스킹(프롬프트 + 후처리 정규식 이중)
6. **기기 내 저장**: IndexedDB 텍스트만, 사용자 삭제권 보장
7. **공유 옵트인**: 원본 사진 공유는 명시적 체크 시에만
8. **기존 자산 활용**: '바이브코딩 서비스 보안 점검 가이드' 체크리스트를 배포 전 전수 점검에 적용
9. 공공데이터·오픈소스 출처 표기 (KISA, 공공데이터포털, 사용 라이브러리)
